<H3Bordered text="Overview" />

Stores a session token and updates the session associated with the specified session key, or by default the active session.

<ul>
  <li>This function parses and stores a signed JWT session token in local storage, associating it with the given session key.</li>
  <li>If a sessionKey is provided, the session will be stored under that key; otherwise, it will use the default session key.</li>
  <li>If a session already exists for the session key, its associated key pair will be deleted before storing the new session.</li>
  <li>After storing the session, any unused key pairs are automatically cleared from storage.</li>
  <li>Ensures that session management is consistent and prevents orphaned key pairs.</li>
</ul>
